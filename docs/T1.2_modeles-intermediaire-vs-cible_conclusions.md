## Modele intermediaire (sept 2026 → Q3 2027)

### Principe

Deux lignes separees dans la facture client :
1. **Abonnement plateforme** — per-agent, bas prix pour entrer
2. **Service FDE** — facture en forfait mensuel (volume de jours inclus)

### Tiers plateforme

| Tier | Prix | Agents | Users | Mode deploiement |
|------|------|--------|-------|-----------------|
| **Starter** | 1 500 - 2 000 EUR/agent/mois | 1-2 agents | ~5 users | Cloud mutualise |
| **Growth** | 2 500 - 3 000 EUR/agent/mois | 3-5 agents | ~20 users | Cloud mutualise |
| **Enterprise** | Forfait negocie (200-300k/an) | Illimite | Illimite | Deploiement personnalise (on-premise ou cloud dedie) |

Contrainte respectee : abonnement total < 10k/mois (ex : Bouygues entre a ~3 500 EUR/mois pour 2 agents).

### Service FDE

- **Atelier decouverte** : 20 000 EUR (one-shot)
- **Deploiement par use case** : 40 000 EUR (one-shot)
- **Amelioration continue** : forfait mensuel avec volume de jours inclus
  - Base : ~5 jours/mois/use case = 5 750 EUR/mois = ~70k/an
  - Au-dela du forfait : regie a 1 150 EUR/jour
  - Variabilite forte : certains use cases quasi-automatises (peu de jours), d'autres complexes (beaucoup de jours)
  - **Moyenne BP retenue** : ~50k/an par use case (approximation, a affiner avec T2.2)

### Ratio FDE par use case (hypothese BP)

- 1 FDE pour ~4 use cases en moyenne
- Les FDEs font aussi les ateliers (pas uniquement le run)
- Commencer a 3-4-6 FDEs des le depart (pas 2)

## Modele cible (Q3 2027+)

### Principe

Le client voit **un seul prix** (plus de separation service/plateforme).

Deux composantes de facturation, combinables :

### Composante 1 — Credits (standard, base du modele)

Systeme de credits prepaye avec marge ~30% sur le cout modele + infra.

**Grille de reference :**

| Pack | Credits/mois | Prix/mois | Prix unitaire |
|------|-------------|-----------|---------------|
| Starter | 10 000 credits | 150 EUR | 0.015 EUR/credit |
| Growth | 50 000 credits | 600 EUR | 0.012 EUR/credit |
| Enterprise | Custom | Custom | Negocie |

*1 credit ≈ 1k tokens (a affiner selon les modeles utilises)*

Marge : on achete les tokens au prix modele + infra, on revend avec ~30% integre dans le prix du credit.

### Composante 2 — Fee de succes par case (option premium)

- **Principe** : fee fixe declenchee quand un case passe en statut "done" dans la plateforme
- **Avantage** : parfaitement aligne avec la valeur delivree, message VC fort, differenciateur vs concurrence
- **Propose en option** sur certains use cases simples a mesurer (pas par defaut)

**Grille de reference :**

| Use case | Fee par case resolu |
|----------|-------------------|
| Facturation / rapprochement | 0.50 - 2 EUR par facture traitee |
| Service client | 1 - 5 EUR par ticket resolu |
| Saisie / execution | 0.20 - 1 EUR par transaction |

**Definition du succes** : case passe en statut "done" dans la plateforme (tracked nativement dans le data model). La definition exacte est contractualisee par use case.

### Place du FDE dans le modele cible (Option C actee)

| Segment | FDE | Facturation |
|---------|-----|-------------|
| ETI (self-service) | Pas de FDE | Credits + option success fee uniquement |
| Grand compte | FDE integre au package | Inclus dans le forfait, non visible |
| Tres grand compte | FDE integre ("customer success engineer") | Inclus dans le forfait enterprise, un seul prix |

### Amelioration continue dans le modele cible

- **ETI self-service** : faite par le client via le SDK (roadmap M5 — fxpkg CLI + SDK docs)
- **Enterprise** : incluse dans le package (FDE integre = customer success engineer)

## Bascule entre les deux modeles

### Timeline par segment

| Segment | Bascule modele cible | Deblocage |
|---------|---------------------|-----------|
| ETI (~100M CA) | Q3 2027 | RGPD + DPA solide + hebergement EU (suffisant contractuellement maintenant) |
| Grand compte (~1Md CA) | Q4 2027 / Q1 2028 | ISO 27001 + DPA + audit securite (~12-18 mois) |
| Tres grand compte (~40-50Md CA) | 2028+ | SecNumCloud ou partenariat OVH |

**Note** : ISO 27001 est le vrai deblocage pour les GC. A lancer maintenant si pas en cours (12-18 mois pour obtenir la certification).

### Mecanique de transition client

Standard du marche : **grandfathering avec migration incitee**

1. Clients sur modele intermediaire grandfatheres 6-12 mois apres lancement du modele cible
2. Migration anticipee proposee avec incentive (2 mois offerts ou credits bonus)
3. A la date de renouvellement du contrat : nouveau modele applique automatiquement (preavis 3 mois)
4. Aucune migration forcee sans accord du client

**Impact BP** : clients signes 2026 - debut 2027 basculent naturellement lors de leur renouvellement (fin 2027 / debut 2028).

## Decisions actees

| # | Decision |
|---|----------|
| 1 | Modele intermediaire = 2 lignes (abonnement per-agent + service FDE forfait) |
| 2 | Tiers : Starter (1.5-2k/agent/mois) / Growth (2.5-3k/agent/mois) / Enterprise (forfait negocie) |
| 3 | Differenciateur enterprise = deploiement personnalise (on-premise ou cloud dedie) |
| 4 | Service FDE en transition = forfait mensuel (~5j inclus/use case), regie au-dela |
| 5 | Modele cible = credits (standard) + success fee (option premium) |
| 6 | Marge credits : ~30% sur cout modele + infra |
| 7 | Success fee = case passe en statut "done" (tracked plateforme), contractualise par use case |
| 8 | FDE dans modele cible = cout de delivery integre au package enterprise (pas un produit) |
| 9 | ETI self-service = amelioration continue via SDK client |
| 10 | Bascule progressive par segment (ETI Q3 2027 → GC Q4 2027 → TGC 2028+) |
| 11 | Conformite deblocante : RGPD/DPA pour ETI, ISO 27001 pour GC, SecNumCloud/OVH pour TGC |
| 12 | Transition client = grandfathering + migration incitee au renouvellement |
| 13 | SLA = custom par client pour l'instant |
